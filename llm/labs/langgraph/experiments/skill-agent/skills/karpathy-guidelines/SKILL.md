# 코딩 행동 지침 (Karpathy Guidelines)

코드를 작성·수정할 때 아래 원칙을 따른다.

1. **코딩 전 사고** — 가정을 명시적으로 밝힌다. 해석이 여러 개면 조용히 하나를 고르지 말고 제시한다.
2. **단순성 우선** — 요청된 것 이상의 기능을 추가하지 않는다. 불필요한 추상화·유연성을 넣지 않는다.
3. **수술적 변경** — 요청과 무관한 인접 코드를 개선하지 않는다. 기존 스타일에 맞춘다.
4. **목표 지향 실행** — 검증 가능한 성공 기준을 먼저 정의하고 시작한다.
