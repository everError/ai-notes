# VLM vs VDR 정리

## 한 줄 요약

- **VLM**: 이미지를 **이해해서 텍스트를 생성**하는 모델
- **VDR**: 이미지를 **임베딩해서 검색**하는 모델

둘 다 "이미지+텍스트를 다룬다"는 점은 같지만, **출력**이 다릅니다.

---

## VLM (Vision-Language Model)

**정의:** 이미지와 텍스트를 함께 입력받아 **텍스트를 생성**하는 멀티모달 LLM. MLLM(Multimodal LLM)이라고도 부름.

**입출력:**

```
입력: 이미지 + "이 그래프에서 Q3 매출이 얼마야?"
출력: "Q3 매출은 200억원으로 보입니다."
```

**핵심 능력:**

- 이미지 캡셔닝 (사진 보고 설명 생성)
- VQA (이미지에 대한 질문 답변)
- OCR-free 문서 읽기 (영수증, 표, 차트 이해)
- 멀티모달 추론 (이미지 보면서 추론)

**대표 모델:**

| 모델                  | 만든 곳         | 특징                     |
| --------------------- | --------------- | ------------------------ |
| GPT-4V / GPT-4o       | OpenAI          | 클로즈드, 최상위 성능    |
| Claude 3/4            | Anthropic       | 클로즈드, 문서 이해 강함 |
| Gemini                | Google          | 클로즈드                 |
| Qwen2-VL / Qwen2.5-VL | Alibaba         | 오픈, 다국어 강함        |
| LLaVA                 | 학계            | 오픈, 연구용             |
| PaliGemma             | Google          | 오픈, 경량               |
| InternVL              | Shanghai AI Lab | 오픈                     |

**아키텍처 (개념):**

```
이미지 → Vision Encoder (보통 ViT) ─┐
                                    ├─→ LLM 디코더 → 텍스트 생성
텍스트 → Text Tokenizer ────────────┘
```

**언제 쓰나:**

- "이 사진에 뭐가 있어?" 같은 질문 답변
- 차트/표를 읽고 설명
- 영수증 자동 분류
- UI 스크린샷 분석
- RAG에서 캡션 자동 생성용

---

## VDR (Visual Document Retrieval)

**정의:** 문서 이미지를 임베딩 벡터로 변환해서 **유사도 기반 검색**에 쓰는 모델. 텍스트를 생성하지 않음.

**입출력:**

```
입력: PDF 페이지 이미지
출력: 패치별 멀티벡터 [vec₁, vec₂, ..., vecₙ]
        ↓
      벡터DB에 저장 → 쿼리로 검색
```

**핵심 능력:**

- OCR/캡셔닝 없이 PDF 페이지를 이미지째로 임베딩
- 표·차트·다이어그램의 시각적 구조 보존
- 멀티벡터 late interaction으로 정밀 매칭

**대표 모델:**

| 모델                    | 백본 VLM   | 특징                    |
| ----------------------- | ---------- | ----------------------- |
| ColPali                 | PaliGemma  | VDR의 시초, 2024        |
| ColQwen2                | Qwen2-VL   | ColPali의 후속, 더 강력 |
| ColQwen2.5              | Qwen2.5-VL | 최신                    |
| Jina Embeddings v4      | Qwen2.5    | 상용 옵션               |
| Nemo Retriever ColEmbed | Eagle2     | NVIDIA                  |

**아키텍처 (개념):**

```
이미지 → Vision Encoder → 패치별 임베딩 보존 ─→ 멀티벡터 출력
                          (LLM 디코더 없음, 또는 임베딩만 추출)
```

VLM과 다른 점은 **LLM 디코더가 없거나, 있어도 텍스트 생성에 안 쓰고 임베딩만 뽑아낸다**는 것.

**언제 쓰나:**

- PDF가 많은 RAG 시스템 (재무제표, 논문, 매뉴얼)
- 표·차트가 핵심인 문서 검색
- OCR 품질이 안 좋은 다국어/수식 문서
- "이 차트와 비슷한 페이지 찾아줘" 같은 시각 검색

---

## 핵심 비교표

| 항목                     | VLM                  | VDR               |
| ------------------------ | -------------------- | ----------------- |
| **출력**                 | 텍스트               | 벡터 (임베딩)     |
| **목적**                 | 생성·이해·답변       | 검색·매칭         |
| **GPU 메모리 (추론)**    | 큼 (LLM 디코더 포함) | 작음 (인코더만)   |
| **출력 단위**            | 토큰 시퀀스          | 패치별 멀티벡터   |
| **벡터DB 필요?**         | ❌                   | ✅                |
| **챗봇으로 쓸 수 있나?** | ✅                   | ❌                |
| **백본**                 | ViT + LLM            | VLM의 인코더 부분 |
| **대표**                 | GPT-4V, Qwen2-VL     | ColPali, ColQwen  |

---

## 둘이 어떻게 같이 쓰이나

전형적인 **멀티모달 RAG 파이프라인**:

```
① 인덱싱 단계
   PDF → 페이지 이미지 → VDR (ColQwen) → 멀티벡터 → Milvus 저장

② 검색 단계
   사용자 질문 → VDR로 쿼리 임베딩 → Milvus 유사도 검색 → 후보 페이지

③ 생성 단계
   후보 페이지 이미지 + 질문 → VLM (Qwen2-VL) → 최종 답변
```

즉:

- **VDR이 "찾아오고"** (retriever)
- **VLM이 "답변한다"** (generator)

이 둘이 한 쌍입니다. ColQwen + Qwen2-VL 조합이 자연스러운 이유는 백본이 같은 패밀리라서 임베딩 공간이 잘 맞아서이기도 합니다.

---

## 헷갈리기 쉬운 부분 정리

**Q1. ColQwen은 Qwen2-VL을 쓰는데 왜 VLM이 아니야?**
→ Qwen2-VL을 백본으로 빌려왔지만, 학습/사용 목적이 텍스트 생성이 아니라 임베딩 추출이라서 정체성이 VDR입니다. 자동차 엔진을 발전기에 갖다 써도 그건 자동차가 아니라 발전기인 것과 비슷합니다.

**Q2. CLIP은 VDR이야?**
→ CLIP은 **이미지 임베딩 모델**이지만 단일 벡터(per-image)이고 일반 이미지(사진) 대상입니다. VDR은 **문서 이미지** 특화 + **멀티벡터** 방식이라 구분합니다. 굳이 분류하면 CLIP은 "image embedder", ColQwen은 "VDR".

**Q3. VLM으로도 검색이 되지 않나?**
→ 됩니다. VLM에 "이 페이지가 매출 차트인가?"라고 물어보면 답변하고, 그 답을 검색에 쓸 수 있습니다. 하지만 페이지마다 VLM 추론을 돌려야 해서 **수만 페이지면 비용·속도 문제**가 큽니다. VDR은 한 번 임베딩하면 그 다음부턴 벡터 비교만 하면 되어서 훨씬 효율적입니다.

**Q4. VLM은 대형, VDR은 소형인가?**
→ 일반적으론 그렇지만 절대적인 건 아닙니다. ColQwen2.5-7B 같은 큰 VDR도 있고, 작은 VLM도 있습니다. 다만 같은 파라미터 수라면 VLM 추론이 더 무겁습니다 (디코딩 단계 때문).

---

요약하면 **VLM은 "보고 말하는 모델", VDR은 "보고 색인하는 모델"** 입니다. 같은 비주얼 백본을 공유하지만 출력단의 설계와 용도가 완전히 다릅니다.
